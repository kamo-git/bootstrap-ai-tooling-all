## 2025-12-22
### 変更概要
- コーディング前後のテスト実施ルールを追加

### 変更理由
- コーディング前に影響範囲を調査し、テストを作成・実行することで、変更による影響を事前に把握できるようにするため
- コーディング後に同じテストを再実行することで、変更が正しく動作することを確認するため
- テスト駆動開発の原則を導入し、品質向上と早期バグ発見を実現するため

### 影響範囲
- 対象ドキュメント：
  - docs/ai/PROJECT_CONTEXT.md（セクション9. テストの義務）
  - .github/copilot-instructions.md
  - AGENTS.md
  - CLAUDE.md
  - .coderabbit.yaml（テストの義務セクション）
  - .github/pull_request_template.md（影響範囲調査・テスト実施項目を追加）
- 既存データへの影響：なし（ルール追加のみ）
- 運用への影響：コーディング前にテスト作成・実施が必須となる

### ロールバック観点
- 該当セクションを元の内容に戻せば復旧可能
- 既存のテスト義務ルールは維持されており、より厳格化された内容
